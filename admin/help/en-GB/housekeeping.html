Housekeeping

Description

This screen offers a set of tools to keep your database clean from unused data and images, and to trigger the Autoarchive function.

Tip

Remove unnecessary images regularly.

Navigation

To access this screen use: Components → JEM → Housekeeping

User Access Level

Super Administrator

Toolbar Icons

Back: Takes you back to the JEM Main menu.

Help: Clicking the Help icon opens a related help screen.

Main area

Remove images: Here you can clean up unused images in your database with a simple click of the icon. This action extracts all used images from the database and then compares the result with all available images. Images which are not used are deleted immediately. Note that this feature should be used only if the JEM images are not used elsewhere (such as content items in other articles).

Trigger Autoarchive: Normally the Auto-archive function is triggered when the date changes to a new day and a Frontend visitor visits a JEM view.
The Auto-archive function performs this action:
1. The past events are archived or deleted, according what rule is set in Settings → Basis Settings → Event handling.

Delete all JEM data: Deletes all JEM data in the Database, including events, venues, categories etc. You can use this feature for instance when you have installed the sample data at installation to test JEM before using it, or when you have entered your own data to get acquainted to the system first, before actually using it.
